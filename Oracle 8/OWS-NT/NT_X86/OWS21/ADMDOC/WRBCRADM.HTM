Configuring WRB Cartridges

The WRB Cartridge Administration Form allows you to configure the WRB's behavior for the selected Web Listener process. Before using this form, you must go to the Web Request Broker Administration Form, select a Web Listener process from the list, and click its Configure button. This takes you to the WRB Cartridge Administration Form.

This form allows you to copy The WRB configuration for the selected Web Listener process from another Web Listener process, or modify or delete the selected Web Listener's WRB configuration. It also allows you to configure a new WRB cartridge or modify an existing WRB cartridge for the selected Web Listener process.

Copying Another Listener's WRB Configuration

 If you're configuring a new Web Listener, you might want to initialize its WRB configuration from another Web Listener process. To do this, enter the Listener name from which you want to copy in the Copy WRB Configuration From Listener field and click the Copy WRB Configuration button. This initializes the form's fields from the specified Web Listener's WRB configuration. You can then modify the displayed configuration and click the Modify Configuration button at any time to apply your changes.

Configuring a New WRB Cartridge

 To configure a new WRB cartridge for the selected Web Listener, you must enter your new cartridge name in the Configure a New WRB Cartridge field and click the Create Cartridge button. This takes you to the WRB Cartridge Modification Form, where you can define configuration parameters for your new WRB Cartridge.

To make your new WRB cartridge accessible to the selected Web Listener, you must also add at least one entry for your new cartridge to both the Applications and Objects and Applications and Directories sections of the WRB Cartridge Administration Form.

Modifying an Existing WRB Cartridge

 To modify an existing WRB cartridge configuration, you can change its entries in the Applications and Objects and Applications and Directories sections of the form. You can also select the WRB cartridge from the list on the form and click its Configure button to modify configuration parameters specific to that WRB cartridge.

WRB Configuration Parameters

 The WRB Cartridge Administration Form is divided into two groups of configuration parameters:

  • Applications and Objects--This section lists all the WRB cartridges that the WRB Dispatcher can run. You must add an entry in this section for any new WRB cartridge you develop to make it accessible to the selected Web Listener.
  • Applications and Directories--This section defines virtual pathnames to cartridge shared libraries, scripts, data files, and other support files specific to each WRB cartridge.

Applications and Objects

 This section lists all the WRB cartridges that the WRB Dispatcher can run. Each entry in this section specifies the following information for a particular WRB cartridge:

App

A short alphanumeric name identifying a WRB cartridge--this is the name entered in the Configure a New WRB Cartridge field when the cartridge was created.

Object Path

The local file system path to the WRB cartridge shared library.

Entry Point

The name of the entry point routine in the WRB cartridge shared library.

Min

The minimum number of WRB Execution Engines (WRBXs) running this WRB cartridge that must be running at all times.

Max

The maximum number of WRBXs running this WRB cartridge that may run at the same time.

Thread(T/P)

Thread model for Oracle Cartridge services. T for Thread model and P for Process model.

Applications and Directories

 This section defines virtual pathnames to cartridge shared libraries, scripts, data files, and other support files specific to each WRB service. If you're configuring a new WRB cartridge that allows request URLs to refer such files, you must add one or more entries in this section to define virtual pathnames to these files as you want clients to refer to them.

The entries in this section combine with the entries in the Applications and MIME Types section to determine how and when the selected Web Listener dispatches requests to WRB cartridges.

The format of this section is similar to the Directory Mappings section of the Oracle Web Listener Advanced Configuration Form. Each entry must specify the following information:

Virtual Path

A virtual name by which request URLs can refer to the physical (file-system) directory containing cartridge shared libraries, scripts, data files, or other support files.

App

The name of the WRB cartridge as it appears in the App field of the cartridge entry in the Applications and Objects section.

Physical Path

The name of a file-system directory containing cartridge shared libraries, scripts, data files, or other support files.

Protecting Applications

 This section allows you to assign authentication and/or restriction schemes to protect individual files and directories used by WRB cartridges. Each entry in this section has the following fields:

Virtual Path

A virtual filename, directory name, or file pathname incorporating a wildcard. Virtual pathnames for each WRB cartridge are defined in the Applications and Directories section of the form.

Basic/Digest

A pull-down menu that specifies whether to apply basic authentication, digest authentication, or neither to the specified files or directories. If you use this field to specify an authentication scheme, you must specify a realm in the Realm field.

Realm

For authentication schemes, specifies one of the realms defined in the Basic or Digest Authentication section.

&/|

This pull-down menu allows you to assign both an authentication and a restriction scheme to the specified files or directories.

If you choose `&,' requestors are required to access the WRB cartridge from the specified address group and provide a username and password. If you choose `|,' requestors may access the specified files or directories if they satisfy one of the two schemes.

IP/Domain

A pull-down menu that specifies whether to apply IP-based restriction, domain-based restriction, or neither to the specified files or directories. If you use this field to specify a restriction scheme, you must specify an address group in the Group field.

Group

For restriction schemes, specifies one of the address groups defined in the IP-based or Domain-based Restriction sections.

Applications and MIME Types

 This section associates file extensions that identify Multipurpose Internet Mail Extensions (MIME) types with WRB cartridges. This section combines with the Applications and Directories section to determine how and when the selected Web Listener dispatches requests to WRB cartridges.

For the selected Web Listener to dispatch a request to a particular WRB cartridge, the request URL must:

  • specify a virtual path associated with the WRB cartridge by the Applications and Directories section
  • specify a filename extension associated with the WRB cartridge by the Applications and MIME Types section (this section)
If only one or none of these criteria is satisfied, the Web Listener handles the request itself.

Application

The name of the WRB cartridge as it appears in the Applications and Objects section.

+/-

This pull-down menu has two states. If you choose `+' (the default), only request URLs that use the specified extensions may be dispatched to the specified application (cartridge). If you choose `-,' only request URLs that do not use the specified extensions may be dispatched to the specified application.

File Extension(s)

A comma-separated list of filename extensions that identify MIME types.
