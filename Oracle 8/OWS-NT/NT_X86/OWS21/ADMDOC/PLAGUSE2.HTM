Invoking the PL/SQL Agent from a URL

In general, PL/SQL Agent applications are invoked just as other applications are. See Invoking an Application. The following features are specific to the PL/SQL Agent:

  • Passing Multivalued Parameters (PL/SQL Tables) to the PL/SQL Agent.
  • Overloading PL/SQL Agent Procedures and Functions.

Passing Multivalued Parameters (PL/SQL Tables) to the PL/SQL Agent

Some HTML variables can have multiple values, such as form fields that allow multiple selection. The HTML tag "SELECT" allows users to select values from a given set. In this case, you can pass the multiple values to a single PL/SQL table. Among other things, this makes it easy for you to be flexible about how many values the user can pick, and it makes it easy for you to process her answers as a unit.

When you pass a group of values to a PL/SQL table, they are each stored in subsequent rows of the table. For this reason, you must pass all of the values together. Since the rows of a PL/SQL table are identified only by sequential numbers, you must also preserve the order of the values, if that order is significant. This is an exception to the general rule that you need not be concerned with the order in which PL/SQL parameters are given in the URL or the HTTP header.

Here are some links to PL/SQL Web page samples that use PL/SQL tables to pass multivalued parameters.

  • The QUERY_FORM PL/SQL Web Page Example prints an HTML page using all the columns of a specified table. It is also a good example of how to use PL/SQL code to create Web pages.
  • The DO_QUERY PL/SQL Web Page Example is a variation on QUERY_FORM that let's the user choose which columns to include in the query.
As these examples show, if you cannot guarantee that at least one value will be submitted for the PL/SQL table, it is a good idea to use a hidden place-holder variable as the first value. This is because you cannot provide a default NULL value for a PL/SQL table and therefore have no easy way to deal with the empty set. If it is legitimate for a user to select zero options, you have to handle that case specially. Not providing a value for the PL/SQL table will produce an error.

Note that PL/SQL tables used as parameters to the PL/SQL Agent must have a base type of VARCHAR2. However, Oracle can convert VARCHAR2 to many other datatypes, such as NUMBER, DATE, or LONG. The maximum length of a VARCHAR2 value is 32K (32767 bytes).

Overloading PL/SQL Agent Procedures and Functions

PL/SQL allows you to overload procedures and functions that are in PL/SQL packages (standalone procedures and functions cannot be overloaded). In overloading, multiple procedures or functions have the same name, but are distinguished by the fact that they take different parameters. Normally, in PL/SQL, the matching parameters of overloaded procedures can be distinguished by having different datatypes. Since HTML data is not associated with datatypes, however, WebServer requires that matching parameters have different names.
