Establishing a Database Connection Using DCDs

DCDs are files that contain information the PL/SQL Agent uses to connect to the database. The DCD is specified as part of the URL in a HTTP request. This enables you to run the same application under different privileges by using different DCDs. For an example of how a DCD is specified in a URL, see Example of PL/SQL Agent Parameter Passing Using GET. This page describes the content of the DCD.

Generally, you should use WebServer Manager to create the DCDs as described in PL/SQL Agent Administration. If necessary, you can create or edit the files directly; they are ASCII text.

DCD Contents

The fields of the DCD are listed below. Click on any field to move to its description.

  • username
  • password
  • ORACLE_HOME
  • ORACLE_SID
  • SQL*Net V2 Service Name or Connect String
  • owa_err_page
  • owa_valid_ports
  • owa_log_dir
  • owa_nls_lang
username
All SQL and PL/SQL statements are executed by the database under the auspices of some database user. This name identifies that user. The username determines the schema (logical section of a database) that the PL/SQL Agent connects to, the actions it can perform, the resources (disk space and so on) it can use, and the level of monitoring of its activities that the Oracle RDBMS performs. For more information, see "Overview of the Oracle7 Server, SQL, and PL/SQL" in the Introduction to the Oracle WebServer.

password
This is provided if you want to password protect the DCD, ensuring that only certain people connect as the user identified by username. Keep in mind, however, that the users who connect to the database as username still will be able to do only what the application gives them the ability to specify through URLs.

ORACLE_HOME
This is the root of the Oracle7 Server code tree in the OS file system.

ORACLE_SID
The name of the Oracle database to which the PL/SQL Agent connects, if it is a local database.

SQL*Net V2 Service Name or Connect String
The name of the Oracle database to which the PL/SQL Agent connects, if it is a remote database.

owa_err_page
This is the pathname of the HTML document that the PL/SQL Agent is to return to the client's browser when an system error occurs. This is the actual path as understood by the OS, not the virtual path as configured in the Web Listener. If this parameter is omitted, a default page is sent. For more information on system and application errors in the PL/SQL Agent, see Oracle PL/SQL Agent Error Handling.

owa_valid_ports
The valid Web Listener network ports to which the PL/SQL Agent will respond. The network port is, of course, specified in the URL. You must make sure that the URLs your applications generate provide port numbers that are valid for the WRB Cartridge or CGI program they specify.

owa_log_dir
The directory where the PL/SQL Agent writes its log status file. This file provides general status and detailed error information for use by the WebServer Administrator.

owa_nls_lang

The NLS_LANG of the Oracle7 database to which the PL/SQL Agent connects. If not specified, the PL/SQL Agent administration program looks up the database NLS_LANG when the HTTP request is submitted. The NLS_LANG indicates the language to be used. Here is an example:

AMERICAN_AMERICA.US7ASCII
