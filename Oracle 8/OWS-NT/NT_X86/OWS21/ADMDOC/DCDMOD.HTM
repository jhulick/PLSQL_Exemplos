Modifying a Database Connection Descriptor

The Oracle Database Connection Descriptor Modification Form allows you to:

  • Create a new Database Connection Descriptor (DCD) by copying the configuration of an existing DCD.
  • Modify the configuration of an existing DCD.
  • Delete a DCD.
To use this form, you must have selected a DCD name from the list on the Oracle PL/SQL Agent Administration Form and clicked its Configure button. This takes you to the Oracle Database Connection Descriptor Modification Form.

Copying a DCD

 To create a new DCD based on an existing DCD configuration, select the existing DCD, and go to the Oracle Database Connection Descriptor Modification Form. On this form, change the value of the DCD field to the name you choose for the new DCD and click the Copy Service button at the bottom of the form. The original DCD is unchanged, and the Oracle Database Connection Descriptor Modification Form now shows the configuration of the new DCD.

Making Your DCD Accessible

 Once you've created a new DCD, you must make it accessible to at least one Web Listener process. To do this, you must make sure that the Web Listener defines a virtual path to the owa executable. For the following URL to work, for example, the specified Web Listener would have to have a virtual directory
/hr mapped to the file-system directory containing the owa executable:

http://listener-name/hr/owa/hr_proc.lookup

To make your new DCD accessible to a Web Listener process:

  1. Go to the Oracle Web Listener Administration Form.

  2. Select a Web Listener process from the list and click its configure button to take you to the Oracle Web Listener Advanced Configuration Form

  3. Go to the Directory Mappings section of the form and define a virtual directory mapping to make your DCD accessible to the selected Web Listener process.

  4. Repeat this process for each Web Listener you want to have access to this DCD.

Controlling Access to Your DCD

 You can use the Web Listener's security features to restrict access to your DCD and/or require clients to authenticate themselves before using it:

  1. Go to the Oracle Web Listener Administration Form.

  2. Select a Web Listener process from the list and click its Configure button to take you to the Oracle Web Listener Advanced Configuration Form

  3. Follow the link to the Oracle Web Listener Configure Security Form and follow the instructions.

  4. Repeat this process for each Web Listener that uses the DCD.

Modifying a DCD

 At any time, you can click the Modify Service button to apply changes you've made to the configuration parameters of the selected DCD. If you change the DCD name, the DCD is renamed, not copied.

Deleting a DCD

 To delete the selected DCD, click the Delete Service button.

Configuration Parameters

 The following sections describe the form's fields:

DCD name

 The name of the selected DCD. This name is used in URLs to invoke the DCD. For example, if you had a DCD named HR, and a PL/SQL procedure named hr_proc.lookup, the URL used to execute the procedure would look like this:

http://server-name/hr/owa/hr_proc.lookup

To make the DCD accessible, you must make sure that the Web Listener defines a virtual path to the owa executable. For the example above, there would have to be a virtual directory /hr mapped to the file-system directory containing the owa executable.

If you change the DCD field, you must make sure there is a virtual directory mapping that makes your DCD accessible. See Making Your DCD Accessible.

You can also restrict access to your DCD and/or set it up to require access authentication. See Controlling Access to Your DCD.

DCD names are not case sensitive. The form informs you if the name you specify is already being used.

PL/SQL Agent Database User

 The User ID that the DCD uses to access the database.

If you enter a username that is not yet known to the database, you must click the Create PL/SQL Agent Database User checkbox near the bottom of the form. If the database is a remote database, you must also provide a known database user name and password in the DBA Username and Password fields at the bottom of the form. The PL/SQL Agent uses this user name to connect to the database and create the new username.

Identified By

These radio buttons specify where the password for this database username is stored.

Clicking the Operating System radio button indicates that the database username is also an operating system username, and its password is stored in the system password file.

Clicking the Password radio button indicates that the password for this username is specified in the User Password field.

User Password

The login password for this User ID if the Password radio button is checked.

Confirm Password

Repeat the User Password in this field to make sure you've typed it correctly.

ORACLE_HOME

 The directory where the oracle database is installed. For a remote database, specify the ORACLE_HOME of the database on the remote host.

Identifying the Database

 You must identify the database that the DCD is to access differently depending on whether the database runs on the same host as the Oracle WebServer or on a remote host. If you accidentally specify both parameters, the ORACLE_SID field is ignored.

ORACLE_SID

(For local database access only) For DCD that accesses a database on the same host as the web server, you must specify the ORACLE_SID of the database with which to connect.

SQL*Net V2 Service

(For remote database access only) For a DCD that accesses a database on a remote host, you must specify a SQL*Net V2 service name used to connect this DCD with a database on a remote host.

Authorized Ports

 The TCP/IP ports that this DCD may use.

Log File Directory

The file-system directory in which the DCD stores its transaction log file.

HTML Error Page

The file-system pathname of an HTML file to be displayed when an error occurs. If you leave this field blank, the PL/SQL Agent returns a default error message to the requestor.

NLS Language

 Language and character set that this DCD uses, for example AMERICAN_AMERICA.US7ASCII. You may leave this field blank, as it's filled in automatically when you create the DCD.

Create PL/SQL Agent Database User

 Click this checkbox if the user name specified in the PL/SQL Agent Database User field is not yet known to the database. If the database is a remote database, you must also provide a known database name and password in the DBA Username and Password fields. The PL/SQL Agent uses this username to connect to the database and create the new username.

Change PL/SQL Agent Database User Password

 Click this checkbox to change the password for the existing database username specified in the PL/SQL Agent Database User field to the value specified in the User Password field. If the Operating System radio button in the Identified By field is checked, this checkbox is ignored.

Install WebServer Developer's Toolkit PL/SQL Packages

 Install the Oracle WebServer Developer's Toolkit PL/SQL packages under the schema of the user specified in the PL/SQL Agent Database User field. See the Oracle WebServer User's Guide for more information about these packages.

DBA Username

 If the username specified in the PL/SQL Agent Database User field is not yet known to the database to which this DCD is to connect, fill in this field with a username that has administration privileges on the database. The PL/SQL Agent uses this database user to connect to the database and create the new username.

Password

 The password for the known database username specified in the DBA Username field.
