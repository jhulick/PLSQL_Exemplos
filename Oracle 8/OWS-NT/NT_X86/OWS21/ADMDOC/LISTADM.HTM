Managing Oracle Web Listener Processes

The Oracle Web Listener is an HTTP engine capable of accepting requests for hypermedia documents from multiple TCP/IP ports. You can run several Web Listener processes at once, with each accepting connections on a different set of ports.

The Oracle Web Listener Administration form allows you to create a new Web Listener process, modify the configuration of an existing Web Listener process, start or stop a Web Listener process, or delete a Web Listener process.

Creating a New Oracle Web Listener

 To create a new Oracle Web Listener, you must first decide whether you want your Web Listener to accept connections on multiple ports, or only one port: click either the single port or multiple ports radio button, and click the Create Listener button. This takes you to the appropriate Create New Oracle Web Listener form.

Modifying an Oracle Web Listener

 To modify the configuration of an existing Web Listener process, select it from the list of existing Web Listener names by clicking the radio button next to its name. If no Web Listeners appear on the form, there are no Web Listeners configured; to create one, see the preceding section.

Once you've selected the Web Listener you want to modify, click the CONFIGURE button. This takes you to the Oracle Web Listener Advanced Configuration form, which allows you to configure any Web Listener parameters.

Starting, Stopping, or Deleting a Web Listener

 To start, stop, or delete a Web Listener process, click on its name in the list on the form, and click the start, stop, or delete button.
