Starting Up and Shutting Down Databases

The Oracle WebServer Database Administration Form allows you to start up and shutdown an Oracle7 database. This is useful for controlling the databases to which the Oracle WebServer connects. For more information about starting up and shutting down databases and to learn more about database administration in general, see the Oracle7 Server Administrator's Guide.

After selecting a database from the list on the form, you can use the buttons below to start up or shut down the database in several ways:

Start-up Options

Normal

 Clicking the Normal button initiates a normal database start-up procedure. Any errors that occur are reported, and the database remains shut down.

Force

 Clicking the Force button initiates an abort operation on a running database and restarts it normally. Use this option on a running database.

Shutdown Options

Normal

 Clicking the Normal button blocks any new connection attempts and performs a normal database shutdown. Any active user sessions are allowed to terminate before shutdown.

Immediate

 Clicking the Immediate button initiates an attempt to shut down the database immediately. Any active user sessions are disconnected, but active calls are allowed to complete before shutdown. All uncommitted transactions are ignored.

Abort

 Clicking the Abort button terminates all active Oracle processes and reclaims the memory used by them. The next start-up operation will attempt to perform instance recovery.
