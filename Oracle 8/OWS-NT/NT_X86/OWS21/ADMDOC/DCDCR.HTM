Creating a New Database Connection Descriptor

The Oracle Database Connection Descriptor Creation Form allows you to create a new Database Connection Descriptor (DCD). After filling in the required information, you can click the Submit New Service button to create the DCD.

The following sections describe the form's fields:

DCD name

 The name of the DCD you are creating, used in URLs to invoke the DCD. For example, if you had a DCD named HR, and a PS/SQL procedure named hr_proc.lookup, the URL used to execute the procedure would look like this:

http://server-name/hr/owa/hr_proc.lookup

DCD names are not case sensitive. The form informs you if the name you specify is already being used.

PL/SQL Agent Database User

 The User ID that the new DCD is to assume.

If the username you choose is not yet known to the database, you must click the Create PL/SQL Agent Database User checkbox near the bottom of the form. If the database is a remote database, you must also provide a known database user name and password in the DBA Username and Password fields at the bottom of the form. The PL/SQL Agent uses this user name to connect to the database and create the new username.

Identified By

These radio buttons specify where the password for this database username is stored.

Clicking the Operating System radio button indicates that the database username is also an operating system username, and its password is stored in the system password file.

Clicking the Password radio button indicates that the password for this username is specified in the User Password field.

OWA User Password

The login password for this User ID if the Password radio button is checked.

Confirm Password

Repeat the User Password in this field to make sure you've typed it correctly.

ORACLE_HOME

 The directory where the oracle database is installed. For a remote database, specify the ORACLE_HOME of the database on the remote host.

Identifying the Database

 You must identify the database that the DCD is to access differently depending on whether the database runs on the same host as the Oracle WebServer or on a remote host. If you accidentally specify both parameters, the ORACLE_SID field is ignored.

ORACLE_SID

(For local database access only) For a DCD that accesses a database on the same host as the web server, you must specify the ORACLE_SID of the database with which to connect.

SQL*Net V2 Service

(For remote database access only) For a DCD that accesses a database on a remote host, you must specify a SQL*Net V2 service name to be used in connecting this DCD with a database on a remote host.

Authorized Ports

 The TCP/IP ports that this DCD may use.

Log File Directory

The file-system directory in which the DCD will place its transaction log file.

HTML Error Page

The file-system pathname of an HTML file to be displayed when an error occurs. If you leave this field blank, the PL/SQL Agent returns a default error message to the requestor.

NLS Language

 Language and character set for this DCD to use, for example AMERICAN_AMERICA.US7ASCII. You may leave this field blank, as it's filled in automatically when you create the DCD.

Create PL/SQL Agent Database User

 Click this checkbox if the user name specified in the PL/SQL Agent Database User field is not yet known to the database. If the database is a remote database, you must also provide a known database name and password in the DBA Username and Password fields. The PL/SQL Agent uses this username to connect to the database and create the new username.

Change PL/SQL Agent Database User Password

 Click this checkbox to change the password for the existing database username specified in the PL/SQL Agent Database User field to the value specified in the User Password field. If the Operating System radio button in the Identified By field is checked, this checkbox is ignored.

Install WebServer Developer's Toolkit PL/SQL Packages

 Install the Oracle WebServer Developer's Toolkit PL/SQL packages under the schema of the user specified in the PL/SQL Agent Database User field. See the Oracle WebServer User's Guide for more information about these packages.

DBA Username

 If the username specified in the PL/SQL Agent Database User field is not yet known to the database to which this DCD is to connect, fill in this field with a username that has administration privileges on the database. The PL/SQL Agent uses this database user to connect to the database and create the new username.

Password

 The password for the known database username specified in the DBA Username field.

Making Your DCD Accessible to the PL/SQL Agent

 Once you've created a new DCD, you must make it accessible to at least one Web Listener process. The procedure for doing this varies depending on whether you are using the CGI or WRB version of the PL/SQL Agent.

Making Your DCD Accessible to the PL/SQL Agent WRB Cartridge

 Make sure that the WRB configuration defines a virtual path to the owa executable. For the following URL to work, for example, the specified Web Listener would have to have a virtual directory /hr mapped to the file-system directory containing the owa executable:

http://listener-name/hr/owa/hr_proc.lookup

To make your new DCD accessible to a Web Listener process:

  1. Go to the Web Request Broker Administration Form.

  2. Select a Web Listener process from the list and click its configure button to take you to the WRB Cartridge Configuration Form

  3. Go to the Applications and Directories section of the form and define a virtual directory mapping to make your DCD accessible to the PL/SQL Agent WRB Cartridge.

  4. Repeat this process for each Web Listener you want to have access to this DCD.

Making Your DCD Accessible to the PL/SQL Agent CGI Program
  1. Go to the Oracle Web Listener Administration Form.

  2. Select a Web Listener process from the list and click its configure button to take you to the Oracle Web Listener Advanced Configuration Form

  3. Go to the Directory Mappings section of the form and define a virtual directory mapping to make your DCD accessible to the selected Web Listener process.

  4. Repeat this process for each Web Listener you want to have access to this DCD.

Controlling Access to Your DCD

 You can use the Web Listener's security features to restrict access to your DCD and/or require clients to authenticate themselves before using it:

  1. Go to the Oracle Web Listener Administration Form.

  2. Select a Web Listener process from the list and click its Configure button to take you to the Oracle Web Listener Advanced Configuration Form

  3. Follow the link to the Oracle Web Listener Configure Security Form and follow the instructions.

  4. Repeat this process for each Web Listener that uses the DCD.
