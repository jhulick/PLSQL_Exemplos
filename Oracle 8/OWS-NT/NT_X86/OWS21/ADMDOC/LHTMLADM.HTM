Configuring the LiveHTML WRB Cartridge

The Oracle Web Listener LiveHTML Administration Form allows you to control the behavior of the LiveHTML WRB Cartridge for the selected Web Listener. To use this form, you must have used the Oracle Web Listener Administration Form to create or select a Web Listener process, and have followed the links to the Oracle Web Listener Advanced Configuration Form. From there, you must have followed the link to the Oracle Web Listener LiveHTML Administration Form.

The LiveHTML WRB Cartridge parses extensions to the HTML file format that can include variable values and execute operating system commands. Files that contain these extensions use a .shtml extension.

The three sections of the LiveHTML Administration Form allow you to configure three aspects of the LiveHTML Cartridge.

Enable LiveHTML

 Click the Enable LiveHTML Processing radio button to allow the selected Web Listener to use the LiveHTML Cartridge. If you click the Disable LiveHTML Processing radio button, the LiveHTML extensions in requested .shtml files are ignored.

Enable .html Extensions

 Click the Enable .html extensions radio button to allow the selected Web Listener to use the LiveHTML Cartridge to parse all files, not just those having extensions listed in the Extension List field. By default, or if you click the Disable .html extensions radio button, the LiveHTML Cartridge parses only files having extensions listed in the Extension List field.

Extension List

 A space-separated list of filename extensions identifying files to parsed by the selected Web Listener using the LiveHTML Cartridge. By default, or if you click the Disable .html Extensions radio button, only files having these extensions are parsed.

Enable exec Tag

 An exec tag embedded in a .shtml document specifies an operating system command to be executed.

Click the Enable processing exec tag radio button to allow the selected Web Listener to use the LiveHTML Cartridge to execute operating system commands. This is a powerful feature for customizing requested hypermedia documents dynamically, but it can also threaten the security of your WebServer machine. If you enable exec tag processing for a Web Listener process, you might also want to assign a file protection scheme to the .shtml files the Web Listener process can access. (To do this, go back to the Oracle Web Listener Advanced Configuration Form and follow the link to the Oracle Web Listener Configure Security Form.)
