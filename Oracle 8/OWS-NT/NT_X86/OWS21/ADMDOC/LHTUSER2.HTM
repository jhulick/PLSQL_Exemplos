LiveHTML Examples

This section provides examples of the use of various LiveHTML tags. Assume your file contains the following include tag:

 <!--#include file="inc1.html"-->

The content of the file incl1.html is as follows:

<p align=center>

<a href="#FileIncludes">File Includes</a> |

<a href="#Date/TimeInserts">Dates/Times</a> |

<a href="#Date/TimeFormatting">Date/Time Formats</a> |

<a href="#DocumentNames/Paths">Doc Names/Paths</a> |

<a href="#FileSizeAndDate">File Size/Date</a> |

<a href="#HostServer">Host/Server Info</a> |

<a href="#Browser">Browser Info</a> |

<a href="#Animation">Server Push Animation</a> |

<a href="#HitCounters">Counters</a> |

<a href="#ReferringURL">Referring URL</a>

</p>

This would result in the following group of links being inserted into your Web page:

File Includes    Dates/Times    Date/Time Formats  Doc Names/Paths

FileSize/Date    Host/Server Info    Browser Info

Server Push Animation   Counters   Referring URL

Date and Time Formats

The following tags impose DATE and TIME formats and then echo that information into the Web page:

<!--#config timefmt="%A, %B %d, %Y, at %I:%M %p"-->

GMT date/time is Friday, August 23, 1996, at 03:14 AM

LOCAL date/time is Thursday, August 22, 1996, at 08:14 PM

Updated on Tuesday, August 13, 1996, at 03:42 AM

Note: Unix strftime settings are used in the #config timefmt= string. Ordinary characters are copied to the document without conversion, so you can insert "on" or "at" or whatever may be useful.

Info about the Present File

Here are some other useful tags:

This document is <!--#echo var="PATH_TRANSLATED"-->

Its virtual path is <!--#echo var="DOCUMENT_URI"-->

translates to: 

This document is /private1/oracle/ows21/sample/ssi/sstest.html

Its virtual path is /sample/ssi/sstest.html

Info about the Other Files

Two other useful directives are #fsize and #flastmod, which allow you to give the size and date of any file on the server-computer rather than just the current document. They work as follows, using file= for relative addressing and virtual= for full path addressing based on your own home directory.

<!--#config sizefmt="bytes"-->

This gives the file size of 'sstest.html' in bytes: 6405 bytes

                                      or

<!--#config sizefmt="abbrev"-->

This gives the file size of 'sstest.html' in kilobytes: 6 Kbytes

One of the best uses of the #fsize directive is to provide your visitor with the size of any graphics files to be downloaded. This is a tremendous timesaver for you, as an HTML author, if you add and change downloadable images a lot, since you never have to look up the sizes and type them in.

Naming the Client's Browser

It is not of particular interest to the web author, but you may reflect back to the person visiting your pages the browser and version that he or she is using to read your pages.

The code Browser: <!--#echo var="HTTP_USER_AGENT"-->

produces the following:

Mozilla/1.1N (X11; I; SunOS 4.1.3 sun4m)

Providing Host and Server Information 

The following line at the top of a document:

Host: <!--#echo var="REMOTE_HOST"--> (<!--#echo var="REMOTE_ADDR"-->) - Server: <!--#echo var="SERVER_NAME"--> (<!--#echo var="SERVER_SOFTWARE"->)

generates the following information:

Host:stingray.us.oracle.com (186.91.622.103)- Server: tse-tse.us.oracle.com

(Oracle_Web_listener2.1/1.20in2)
