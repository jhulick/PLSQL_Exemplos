UPGRADE the Oracle Web Server 2.0 to 2.1


This page will:
  • UPGRADE all the Web Server Listener under the Oracle Web Server 2.0 area to Oracle Web Server 2.1
  • UPGRADE all the Web Server Listener's default Cartridges (such as JAVA, SSI, OWA) to Oracle Web Server 2.1
  • UPGRADE all the PL/SQL Agent's Database Connection Descriptor (DCD) under the Oracle Web Server 2.0 area to Oracle Web Server 2.1 (NOTE: DCD OWA_DBA & OWA_DEFAULT_SERVICE will NOT be upgraded given the fact that they should be created during the Oracle Web Server 2.1 installation

Before you proceed, you MUST:
  • fresh install the Oracle Web Server 2.1 on the same machine where you run the Oracle Web Server 2.0
  • not have defined any Oracle Web Server 2.1 listener or DCD with the same name of what to be upgraded from Oracle Web Server 2.0. The listener or DCD from the Oracle Web Server 2.0 area with name conflicts with Oracle Web Server 2.1 area will NOT be upgraded.

After the upgrade, you SHOULD:
  • use Oracle Web Request Broker Admin Manager 2.1 to update all your upgraded listener's application directory mapping, application setting if necessary.
  • use Oracle PL/SQL Agent Admin Manager 2.1 to update all your upgraded DCD's log file locations if necessary.
  • use Oracle Web Listener Admin Manager 2.1 to update all your upgraded listener's directory mapping, log files locations if necessary and restart your listener.
