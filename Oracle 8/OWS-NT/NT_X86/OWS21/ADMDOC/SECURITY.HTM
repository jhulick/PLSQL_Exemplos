Configuring Web Listener Security

The Oracle Web Listener Configure Security Form allows you to configure security modules and assign them to protect specific files and directories. You can also use the form to make secure the TCP/IP ports on which the Web Listener accepts connections.

At any time you can click one of the Modify Listener buttons on the form to apply any changes you have made to the security configuration parameters. Clicking any Modify Listener button submits the entire form for processing.

Authentication

 The Oracle Web Listener provides two ways in which you can require requestors to authenticate themselves before accessing certain files and directories:

Basic Authentication

 Basic authentication allows you to assign passwords to users, assign users to groups, and define groups of users and groups, called "realms." You can then assign the users, groups, and realms to specific files and directories, requiring requestors to provide a username and password to gain access. Basic authentication sends unencrypted passwords across the network, making this method subject to subversion. Basic authentication is not recommended when security is critical.

Digest Authentication

 Digest authentication is the same as basic authentication except that it sends passwords encrypted across the network in the form of a cryptographic checksum, also called a "digest." You should use this scheme whenever authentication is required, although some older web browsers do not support it. If you need to support these browsers, however, you have to use basic authentication.

Restriction

 The Oracle Web Listener provides two ways to restrict access to certain files and directories:

IP-based Restriction

 Under this scheme, requestors must issues requests for the protected files and directories from specified IP addresses or groups of IP addresses. IP-based restriction is a somewhat more convenient security scheme than basic or digest authentication in that it doesn't require password files to be maintained, but is consequently somewhat less secure, as clever intruders can falsify their IP addresses.

Also, to use this scheme to grant access to users who go through a proxy server to get outside a firewall, you must grant access to the firewall machine itself, which non-selectively grants access to all users who use that proxy server.

Domain-based Restriction

 Domain-based restriction works the same as IP-based restriction except that it specifies symbolic Domain Name Service (DNS) hostnames and domain names rather than numeric IP addresses. This scheme has an advantage over IP-based restriction in that symbolic names are insulated from underlying changes in IP addresses.

Authentication Parameters

 The Oracle Web Listener Configure Security Form allows you to configure the following authentication parameters:

Basic Users

 Each entry in this section has the following fields:

User Name

A symbolic user name in the style of a UNIX user name.

Password

The password for the specified user name.

Basic Groups

Group Name

 A symbolic group name in the style of a UNIX group name.

User(s)

A space-separated list of users belonging to the specified group.

Basic Realms

Realm

 A symbolic name representing a group of users and groups.

Group(s)

A space-separated list of groups and users belonging to the specified realm.

Digest Users, Digest Groups, and Digest Realms

 These parameters are identical in format to the corresponding basic parameters.

Restriction Parameters

 The Oracle Web Listener Configure Security Form allows you to configure the following restriction parameters:

IP-based Restriction

 Each entry in this section has the following fields:

IP Group

A symbolic name of your choice that identifies a group of IP addresses.

IP Address(es)

A space-separated list of IP addresses, which might look like this:

+144.25.19.* +144.25.18.*

The asterisk `*' is a wildcard character that matches any number, and the plus sign `+' grants access to the IP address that follows. The IP group defined by the list above would grant access to all machines on subnets 18 and 19 of 144.25.

A minus sign `-' explicitly denies access to the IP address that follows. Any IP address not included in an IP group is implicitly denied access to files and directories protected by that group.

Domain-based Restriction

 Each entry in this section has the following fields:

Domain Group

A symbolic name of your choice that identifies a group of DNS domains.

Domain Name(s)

A space-separated list of DNS domain names, which might look like this:

+*.us.mycomp.com +*.sp.mycomp.com

The asterisk `*' is a wildcard that matches any character string, and the plus sign `+' grants access to the DNS domain that follows. The domain group defined by the list above would grant access to all machines in the us.mycomp.com and sp.mycomp.com domains.

A minus sign `-' explicitly denies access to the domain name that follows. Any domain not included in a domain group is implicitly denied access to files and directories protected by that group.

Specifying Protection for Files and Directories

 This section allows you to assign authentication and/or restriction schemes to protect individual files and directories. Each entry in this section has the following fields:

Virtual Path

A virtual filename, directory name, or file pathname incorporating a wildcard. (Each Web Listener's virtual file system is defined on the Oracle Web Listener Advanced Configuration Form.) Some examples are:

  • /support/news--Specifies a single virtual filename.
  • /support/partners/--Specifies an entire virtual directory
  • /support/crypt*--Specifies all files in the virtual directory /support/ starting with the characters "crypt."

Basic/Digest

 A pull-down menu that specifies whether to apply basic authentication, digest authentication, or neither to the specified files or directories. If you use this field to specify an authentication scheme, you must specify a realm in the Realm field.

Realm

For authentication schemes, specifies one of the realms defined in the Basic or Digest Authentication section.

&/|

This pull-down menu allows you to assign both an authentication and a restriction scheme to the specified files or directories.

If you choose `&,' requestors are required to access the Web Listener from the specified address group and provide a username and password. If you choose `|,' requestors may access the specified files or directories if they satisfy one of the two schemes.

IP/Domain

A pull-down menu that specifies whether to apply IP-based restriction, domain-based restriction, or neither to the specified files or directories. If you use this field to specify a restriction scheme, you must specify an address group in the Group field.

Group

For restriction schemes, specifies one of the address groups defined in the IP-based or Domain-based Restriction sections.

Securing TCP/IP Ports Using SSL

 In addition to providing protection for individual files and directories, The Oracle Web Listener Configure Security Form allows you to apply the Secure Sockets Layer (SSL) security mechanism to one or more of the TCP/IP ports on which the Web Listener accepts connections.

If you configure the Secure Sockets Layer for a Web Listener using the Oracle Web Listener Configure Security Form, you must go back to the Oracle Web Listener Advanced Configuration Form to apply SSL security to at least on port that the Web Listener uses.

For more information about the concepts involved in SSL, see the Oracle WebServer User's Guide.

The Secure Sockets Layer section of the Configure Security form has the following fields:

Certificate File

 The file-system path to the certificate file that you received from you Certifying Authority. The file must be in one of the following formats:

  • ASCII-64-encoded DER format, having a .der filename extension.
  • PEM certificate approval message format, having a .pem filename extension.
  • PKCS certificate approval message format, having a .pkc filename extension.

Distinguished Name File

 The file-system path to a file generated by the genreq program that contains distinguished name information. The must be in one of the following formats:

  • DER format, having a .der filename extension.
  • PEM format, having a .pem filename extension.

Key File

 The file-system path to a file generated by the genreq program that contains an encrypted version of your RSA private key. The key file must be in one of the following formats:

  • DER format, having a .der filename extension.
  • PEM format, having a .pem filename extension.

Certifying Authority Directory

 The file-system directory where additional certificate files from your Certifying Authority can be added in the future.

Certificate Revocation List Directory

 The file-system directory where additional Certificate Revocation List (CRL) files can be added in the future.
