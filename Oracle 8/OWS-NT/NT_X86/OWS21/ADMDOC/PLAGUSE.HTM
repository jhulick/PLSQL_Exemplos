Using The PL/SQL Agent

This section explains how to use the PL/SQL Agent to write applications that generate dynamic HTML. The PL/SQL Agent can be used either under the WRB (recommended) or CGI.

PL/SQL procedures and functions are stored in the database. The PL/SQL Agent invokes them by issuing commands to the Oracle RDBMS, which then performs the actual execution and sends the output and status messages back to the PL/SQL Agent. Since PL/SQL is actually executed in the database, the first thing the PL/SQL Agent has to do when it receives a request is connect to the Oracle RDBMS. It does this using information from a configuration file called a DCD. The DCD is specified in the URL, along with the specific procedure to be executed and its parameters.

The rest of this section is divided into the following topics:

  • Establishing a Database Connection Using DCDs. This covers the use and content of DCDs.
  • Invoking the PL/SQL Agent from a URL. This covers features of the URL that are specific to the PL/SQL Agent. For information on how URLs invoke applications generally (not PL/SQL specific), see Invoking an Application. The topics covered on this page include the following:
    • Passing Multivalued Parameters (PL/SQL Tables) to the PL/SQL Agent.
    • Overloading PL/SQL Agent Procedures and Functions.
  • Example of PL/SQL Agent Parameter Passing Using GET.
  • Example of PL/SQL Agent Parameter Passing Using POST.
