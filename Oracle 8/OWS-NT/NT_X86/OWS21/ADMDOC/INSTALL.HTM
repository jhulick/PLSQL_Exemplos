Completing Your WebServer Installation

Once you have installed the Oracle WebServer software, you must create and configure three components:

  • An Oracle Web Listener process.
  • A Database Connection Descriptor (DCD) for the administration server.
  • A default DCD.
To complete your WebServer installation, use your web browser to edit the Oracle WebServer Installation form.

Creating a Web Listener

 To create your first non-administration Web Listener, an HTTP engine that can service client requests for hypermedia documents, go to the Creating an Oracle Web Listener section of the form and do the following:

  1. In the Listener Name field, enter an alphanumeric name of no more than six characters that will uniquely identify the new Web Listener process.

  2. In the Host Name field, enter the name that the Web Listener will use to identify itself when processing requests. You should use either the machine's primary hostname, on an alias, such as www.blob.com, by which you want to identify your Web Listener on the Internet. See your operating system documentation to learn how to define a hostname alias.

  3. In the Port Number field, enter the number of the TCP/IP port on which the Listener should accept connections. Port numbers greater than zero and less than 1024 require the Web Listener to run with root privileges. Web Listeners with user privileges may use port numbers between 1024 and 65535.

  4. In the User ID field, enter the name or number of the user whose privileges the Web Listener will assume. Use an operating system user ID, not an Oracle7 username.

  5. In the Group ID field, enter the name or number of the group whose privileges the Web Listener will assume.

  6. Click the Create Listener button.

Note: Beware of assigning privileged user identities such as oracle and group identities such as dba to a Web Listener, because programs it launches to handle browser requests, such as Web Request Broker (WRB) cartridges, inherit the Web Listener's privileges. For security, you might consider assigning your Web Listeners an unprivileged user identity such as nobody.

Creating an Administration DCD

 When you install your WebServer, the installer starts a special Web Listener process, called the administration server, that allows you to run the WebServer Manager to configure and maintain the WebServer. To enable the WebServer Manager's database configuration features, you must define a special Database Connection Descriptor, called the administration DCD.

To create the administration DCD, called OWA_DBA, go to the Creating an Oracle Database Connection Descriptor section of the Oracle WebServer Installation form and do the following:

  1. Click the OWA_DBA radio button.

  2. In the Username field, enter the username of the Oracle database user whose identity the DCD will assume when it connects to the database. Use an Oracle7 username, not an operating system username.

  3. In the Password field, enter a password for this user.

  4. Confirm this password by retyping it in the Confirm field.

  5. If DCD specifies a remote database, and the username you entered is new and has not yet been registered with that database, you must also enter a username that's already registered with the database and a password for that username in the Existing DBA Username and Existing DBA Password fields. The DCD will use this username and password the first time it connects to the database.

  6. If the DCD specifies a local database, enter the directory where the database is installed in the ORACLE_HOME field.

  7. If the DCD specifies a local database, enter its ORACLE_SID in the ORACLE_SID field.

  8. If the DCD specifies a remote database, enter in the SQL*Net V2 Service field the name of the SQL*Net V2 service that OWA_DBA will use to connect to the database.

  9. Click the Create Service button.

Note: You don't need to enter a value in the Port Number field; this value is chosen automatically for OWA_DBA.

Creating a Default DCD

 When a URL that invokes the PL/SQL Agent does not specify a DCD to use, the PL/SQL Agent passes the request to a default DCD called OWA_DEFAULT_SERVICE.

To create the default DCD, go back to the beginning of the form and do the following:

  1. Click the OWA_DEFAULT_SERVICE radio button.

  2. Choose a username and password for the default DCD and enter them in the Username and Password fields; retype the password in the Confirm field. You must choose a username that has access to the database data this DCD will use.

  3. If the default DCD specifies a local database, enter the directory where the database is installed in the ORACLE_HOME field.

  4. In the Port Number field, enter the TCP/IP port number on which the DCD will accept connections.

  5. If the default DCD specifies a local database, enter its ORACLE_SID in the ORACLE_SID field.

  6. If the default DCD will connect to a remote database, enter in the SQL*Net V2 Service field the name of the SQL*Net V2 service that OWA_DEFAULT_SERVICE will use to connect to the database.

  7. Click the Create Service button.
