Using Oracle WebServer 2.1 with the Netscape FastTrack Server

Oracle WebServer 2.1 incorporates architecture enhancements that allow you to run most WebServer components under the Netscape FastTrack server. This page descibes how to configure an existing Netscape FastTrack server installation to use Oracle WebServer components such as the Web Request Broker.

Future revisions of Oracle WebServer will support integration with other popular HTTP servers in addition to the Netscape FastTrack server.

Configuring the Netscape FastTrack Server

The directory structure for your Netscape FastTrack server installation should look like this:

NS_HOME
  bin/
  docs/
  nsapi/
  userdb/
  httpd-server1/
    config/
      magnus.conf
      obj.conf
      mime.types
    logs/
      access
      errors
    start
    stop
  httpd-server2/
    same structure as server1
  start-admin
  stop-admin
  ...

server1 and server2 are the names of currently configured HTTP server (listener) processes. The start and stop files in each server directory are scripts that start and stop the server process.

For each Netscape FastTrack server with which you want to use Oracle WebServer components, you must:

  1. Use your web browser to go to the External Web Listener Registration page and register your Netscape server.

  2. Go to the Web Request Broker Administration page and configure the Web Request Broker for your Netscape server.

  3. cd to $NS_HOME/httpd-server-name/config, where server-name is the name of your Netscape server. Edit the obj.conf file to configure the server to run the NSAPI adapter. Use the file $ORACLE_HOME/ows21/admin/objsamp.conf as a model. Be sure to specify your local directory mappings and substitute your own values for ORACLE_HOME, NS_HOME and svserver-name.app.

  4. Make sure the following files exist:

    • $ORACLE_HOME/ows21/lib/oracle_nsapi.so
    • $ORACLE_HOME/ows21/lib/adp_nsapi.so
    • $ORACLE_HOME/ows21/admin/adp.cfg If these files are not present, there might be a problem with your Oracle WebServer installation. Try reinstalling the WebServer.

  5. Make sure the file $ORACLE_HOME/ows21/admin/svserver-name.app exists, where server-name is the name of your Netscape server. If the file does not exist, you might not have completed steps 1 and 2 successfully. Try repeating these steps.

  6. Run the stop and start scripts in the listener directory to stop and restart your server.
