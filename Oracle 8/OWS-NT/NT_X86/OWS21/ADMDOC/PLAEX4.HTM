OVERLOAD PL/SQL Web Page Example

As explained under Overloading Subprograms in the "Overview of the Oracle7 Server, SQL, and PL/SQL" in the Introduction to the Oracle WebServer, PL/SQL allows you to overload procedures and functions that are in PL/SQL packages (standalone procedures and functions cannot be overloaded). In overloading, multiple procedures or functions have the same name, but are distinguished by the fact that they take different parameters.

Here is a simple example of overloading:

create or replace package overload is

procedure proc1(charval in varchar2);

procedure proc1(numval in number);

end;

create or replace package body overload is

procedure proc1(charval in varchar2) is

begin

htp.print('The character value is '||charval);

end;

procedure proc1(numval in number);

htp.print('The number value is '||numval);

end;

end;

Note: The PL/SQL Agent's support for overloading is slightly more restrictive than that of PL/SQL in itself. Normally, in PL/SQL, two overloaded procedures can take parameters with all of the same names, as long as they differ in datatype. Since the PL/SQL Agent derives its data from HTML, which does not support datatypes, the names of the parameters must be different to distinguish them. For example, consider the following:

create or replace package overload is

procedure proc1(val in varchar2);

procedure proc1(val in number);

end;

If many situations, PL/SQL would be able to distinguish between the two versions of proc1 by whether the parameter passed to the procedure was a number or a varchar2. However, if the val parameter is taken from HTML input, it will have no datatype and the PL/SQL Agent cannot determine which procedure to call. This package will generate an error.

To avoid the error, name the parameters of the procedures differently. For example: val1 for the first version and val2 for the second.
