Oracle Web Listener Advanced Configuration for Listener

On this form are listed most of the parameters with which you can customize your Oracle Web Listener. The only other parameters are found on the Security form and can be linked to below.

At the bottom of each parameter section are three buttons : Copy Listener, Modify Listener, and Delete Listener. You do not need to select a button from a particular section in order for the changes to be made. Selecting any of the buttons will submit the entire form for changes. However, changes to the Security form do need to be submitted from the Security form.

Selecting the Copy Listener button requires that you first change the Listener Name and the port numbers listed in the Network Parameters section on this page. The entire configuration from this page will be submitted as a new Oracle Web Listener, leaving the existing Web Listener intact. The Log Info File, and Error File parameters for the new Web Listener will be set to the default values.

Selecting the Modify Listener button causes the configuration parameters from this page to rewrite the existing Web Listener configuration.

Selecting the Delete Listener button allows you to delete the configuration setup for this Oracle Web Listener.

You must start or restart each listener you configure on this form from the Listener Administration (Listener Home) front page for the changes to take effect.

The global navigation bar is provided both here and at the bottom of the page.

Addresses and Ports

The The Oracle Web Listener supports communication through multiple address/port combinations, each of which is related to a domain name, a base directory and an information log file.

  • The Proxy Switch enables or disables proxying on the current address/port combination. Proxying is off by default.

  • The Address field denotes the IP address on which the Oracle Web Listener listens to requests. Use ANY to tell the Listener to listen on all addresses assigned to the machine. The default is "ANY".

  • The Port field contains the port number on which listens for requests to the corresponding address. The range of available ports is from 1 to 65535. There is no default.

  • The Security field must be set to SSL when the port number in this combination is assigned to be an SSL port. Otherwise, the default value NONE must be used.

  • The Host Name field denotes the name the Listener will use when referring to itself for requests that pass through this combination. There is no default.

  • The Base Directory field denotes the prefix directory name that the Listener will prepend to a URL prior to servicing the request. The default is '/' signifying that the URL will be serviced as is.

  • The Log Information File field is the Log file to which the Listener writes its entries for information messages related to this address/port combination. The default path for the log information file is <OWS20>\log\sv<Listener_Name>.log

Directory Mappings

The Directory Mappings specify the translation of directories from the file system into the Listener's virtual file system. Each line contains one mapping as well as a two-letter flag. The first letter of the flag indicates whether the directory is "normal", that is, contains no scripts (N), contains CGI scripts (C), or contains WinCGI scripts (W). The second letter distinguishes whether the subdirectories should be mapped recursively (R), or non-recursively (N).

The Oracle Web Listener performs Dynamic Directory Rescans, scanning files in a directory as needed. It can recognize and rescan changes when you add, delete, or modify a file in its virtual file system.

The ordering of the directory mappings is important. The parameters in this section must be ordered such that the parent virtual directories are listed before subsequent virtual subdirectories. The first directory mapping in this section must, therefore, reference the virtual root directory (or "/"). When you submit this form, the directory mappings will be sorted (by Virtual Directory) for you. Any mappings with an empty field for File-System Directory or Virtual Directory will be removed.

Be sure to add a trailing slash to each file-system and virtual directory name.

File Cache Definitions

The FileCache section lists files that will be kept open for the life of the running Web Listener. Files can be specified exactly, specified by using the wildcard, '*', or by specifying an entire directory.

Please remember to specify virtual filenames and directories, not actual ones.

Language Extensions

The Language Extensions section specifies the file name extensions which the Web Listener will recognize and which languages and character sets those extensions map to. One or more file extensions can be specified for each language type and character set.

Three technical notes about the values below:

  1. Extensions are case-sensitive.
  2. Language identifiers are described in RFC 1766.
  3. Character sets are described in RFC 1521.
