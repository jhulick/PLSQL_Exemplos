Create and Maintain SQL Objects the Easy Way
With Coder, you can create and edit any server-side objects—tables, stored procedures, and triggers—easily. SQL-Station Coder simplifies the code creation and editing process through code-generation wizards, report wizards, a catalog browser, and an unparalleled editing facility. Coder takes care of syntax and other details, so you can work more productively and deliver robust applications to your users.

With Coder, you can:
  • Execute stored procedures and access output easily
  • Scan and update table data in spreadsheet format
  • View all referential relationships among objects
  • Check in and check out objects to and from any SCC-compliant configuration management tool, including PLATINUM CCC/Harvest and other industry standards
  • Drag and drop table and column names from the Table Lookup Facility to build queries
  • Combine objects into scripts with an advanced scripting facility that automates dependency analysis.
Browse through Multiple RDBMS Catalogs
Having access to multiple database catalogs is a must—especially in today's distributed development environments. SQL-Station's catalog browser maintains multiple concurrent connections to client/server RDBMSs on different platforms—Oracle7, SYBASE and Microsoft SQL Server, and ODBC-compliant databases. You can access any object you need quickly and easily, and work more efficiently. You can even search according to an object's contents. Once you find what you need, you can view, edit, execute, or even debug your object at the click of a button.

Speed through the Editing Process
Now, developers can use an editor built right on top of their database. SQL-Station's editor, based on Premia's CodeWright, was designed to speed up the editing process with advanced features, such as search and replace, drag and drop, syntax highlighting, graphical Grep, bracket matching, API assistance, comment templates, and spell checking—even for scripts larger than 32K. Developers work productively because they can choose among editing modes—Windows Brief, VI, or EMACS.

Use Object Information to Make Better Maintenance Decisions
Coder's report wizards generate information about an object or a group of objects in the format you need it. Object Report Wizards provide valuable information that helps you maintain your database objects. Plus, you can output reports as print or e-mail, or export them to Microsoft Word, Excel, or Lotus 1-2-3.

Improve Application Quality with Maintenance Shortcuts
With SQL-Station, you can maintain high application-quality standards and ensure that your organization's best programming practices are followed by customizing and incorporating them into templates. You can also perform sophisticated maintenance activities. For example, you can drag and drop objects into a compare facility; generate complex scripts, automatically detecting dependencies and ordering scripts accordingly; or even reverse-engineer your database into a script and then alter it.

Execute Your Code
SQL-Station's advanced query and execution facilities are so comprehensive and powerful they can be used independently for quick, periodic database maintenance in addition to procedure testing. SQL-Station lets you rapidly input arguments and execute your code—even SQL-server system procedures.

Debug Server-Side Objects Quickly
The SQL-Station Debugger is powerful, easy to use, and fast. Debugger lets you use familiar client-side debugging techniques—even breakpoints, call stacks, and watch variables—in a Windows environment. The time you'd normally spend on server-side debugging and testing, and the frustration you'd experience in the process, are drastically reduced because Debugger is so powerful.

You can debug, execute, and test any block of PL/SQL code—triggers, procedures, packages, functions, etc. Simply select the object in question, then right click on it to choose the debugging option.

Step through Code to Locate and Fix Bugs
With Debugger, it's easy to locate problem areas and improve application quality. Simply step through code—statement by statement—as Debugger progressively reveals variable values. Once you find a questionable expression, you can assign new variable values then execute it—in Debugger—to evaluate it. Debugger is safe because it creates new versions without altering your production objects. It's also thorough. Debugger handles multiple input and output variable types, supporting anchored data types, and displays complete server output messages. Once you determine that the code in question is a bug, simply drop into Coder's edit window to fix and execute the source code.

Optimize Your Execution Plans
Even if it's bug free, there's still no guarantee that your application will perform at optimum speed and efficiency. Research shows that 60 to 80 percent of application performance problems are caused by untuned SQL statements. With Plan Analyzer, you can slash response times and drastically improve Oracle7's* SQL performance by supporting optimization activities. Plan Analyzer functions like an expert that answers all your questions about SQL tuning. It presents plans to you graphically, with explanations—reaching far beyond Oracle7's performance statistic and tuning options.

With Plan Analyzer, you can:
  • Analyze optimization plans according to your execution environment
  • Capture SQL code that consumes the most resources
  • Find frequently used queries that manipulate data based on attributes without a corresponding index
  • View performance statistics through easily read, graphical presentations
  • Support sophisticated Oracle tuning techniques, such as array binding and hints.
Capture the SQL Code that Consumes the Most Resources
Plan Analyzer's flexible, real-time or batch SQL-capture facility traps SQL code and associated optimization plans from database sources and third-party development tools (such as Visual Basic and PowerBuilder). With the capture facility, you can identify SQL that causes slow transactions based on various filters, such as CPU usage, I/O, elapsed time, and more.

While capturing top offending statements is an important first step, Plan Analyzer also provides optimization expertise. Simply focus on the SQL you need to optimize—Plan Analyzer can take care of the rest.

Empower Expert and Novice Users
Regardless of your experience level, you can perform sophisticated SQL optimization functions. Plan Analyzer guides you through the optimization process intuitively, offering a choice of methods: rule-based, cost-based, or custom methods with hints. It steps you through plans line by line, optionally providing explanations in plain English and animating plans to reveal execution flow. In expert and standard modes, Plan Analyzer makes it easy to choose the right hints. In standard mode, it will even select optimal plans for you.

Let Plan Analyzer Evaluate Your Plan
Plan Analyzer evaluates your plan according to its run-time environment. It then provides a complete list of objects accessed, a detailed statistical analysis of each accessed object, and a list of alternate access paths for each plan. It even color-codes alarm and warning conditions according to the thresholds you define.

Use the Repository for Change Detection and Management
Users can run reports and access or save valuable information in the Plan Analyzer Repository—including current work, previously tuned plans, SQL statements, and performance statistics. If performance suddenly drops, a simple query reveals recent changes, helping you locate the offending SQL quickly.

Over time, Oracle-generated optimization plans, growing table size, and changing indexes will alter your database. Plan Analyzer allows you to run reports to determine how your database has changed. You can find out if, for example, your SQL calls an extinct or rarely used index. Or, you can search for and compare a recently saved plan to a production plan.
